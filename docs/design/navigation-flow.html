页面导航流程

标准顺序：启动说明 → 相册权限 → 宝宝资料 → 扫描 → 结果 → 完成

PAGE 1启动说明页
介绍 App 本地处理、不上传云端的核心理念
●已授权 + 有宝宝数据：冷启动直接跳 PAGE 5 结果页
●引导中途冷启动（权限/资料/扫描三类断点）：从对应中断页面继续，不重走启动引导
+设置入口内增加【重新走新手引导】开关，老用户可主动回本页 NEW
+页面底部小字：「所有图片仅本地设备处理，不上传云端，符合隐私法规」合规
+底部可查看完整隐私协议链接（三处锚点之一：启动页 / 权限页 / 设置页）合规
↓
点击「开始整理」（首次）
PAGE 3相册权限页
先申请权限再录入宝宝信息，符合 Apple 审核规范
+权限弹窗前，页面提前说明「需要相册权限读取宝宝照片，所有数据本地存储不上传」，提升授权率 合规
+部分授权文案优化：「仅可访问部分相册，扫描范围受限，照片筛选结果可能不完整」NEW
●用户从系统设置返回 App → 弹窗友好说明「缺少相册权限无法扫描，可去设置开启或手动传图」，重新触发授权弹窗
●后续去系统关闭权限 → 从任意页面切回 App 弹窗拦截，跳转权限页
+隐私协议锚点（三处之二） 合规
✓ 完全授权

正常进入宝宝资料页，无任何 Banner

⚠ 部分授权

放行进入资料页，全局标记，全链路固定黄色 Banner 透传，不可手动关闭

✕ 拒绝权限

弹窗双选项：
①【去系统设置开权限】
②【手动上传单张照片】（折中方案，降低流失率）NEW

⚠ 部分授权全局状态透传：权限页标记 → 扫描页底部悬浮提示 → 结果页顶部固定 Banner → 保存页提示，全程不丢失，冷启动保留，权限恢复全量后 Banner 自动消失
↓
完全授权 / 部分授权
PAGE 2宝宝资料页
收集宝宝信息，为人脸分组和成长节点做准备
●昵称 必填 — 空值点击「下一步」触发输入框红框 + 悬浮提示「请填写宝宝昵称，用于照片分组」，不禁用按钮 NEW
●生日 可选 性别 可选，保留跳过按钮
+生日/性别字段小字提示：「完善生日可精准筛选对应月龄成长照片」NEW
+新增宝宝中途退出：本地缓存已填信息，下次从结果页进入资料页自动回填 NEW
●结果页 / 保存页提供「新增宝宝」入口，跳转本页，录入后独立触发扫描
↓
昵称已填，点击「下一步」
PAGE 4扫描进度页
读取 → 过滤 → 人脸识别 → 评分 → 生成 Top 20
+右上角常驻【取消】按钮：取消后返回结果页（已有数据）/ 资料页（新宝宝）NEW
●部分授权：底部固定悬浮小字「当前仅获取部分相册，部分照片无法被扫描识别」
●中途切后台 / 杀进程：本地缓存进度，下次进入续扫，不从头重来
+相册为空：展示空页面，双路径按钮：①【前往系统相册添加图片】 ②【手动上传单张照片】NEW
↓
扫描完成
未识别到宝宝照片

弹窗三选项：
①【调整宝宝信息】→ 跳宝宝资料页，改完重新扫描
②【全量扩容扫描】→ 原地全相册重扫
③【取消，返回结果页】→ 保留旧数据 NEW

PAGE 5宝宝 Top 20 结果页
查看 AI 推荐结果，允许用户调整
●部分授权：顶部固定 Banner 不可手动关闭
●功能：移除 / 恢复、预览大图、保存精选
●多宝宝切换：切换时缓存上次筛选状态（移除/精选），不重置
+【新增宝宝】跳转前置校验：若当前权限不全，弹窗提醒「相册权限不全，新增宝宝扫描会缺失图片，建议先开启全量权限」NEW
+全部移除后空状态：展示「暂无宝宝照片」+ 双按钮：①【重新全量扫描】 ②【新增宝宝】NEW
↓
点击「保存精选」
PAGE 6保存完成页
确认保存成功，引导查看 / 分享
●成功：展示已保存数量，按钮：查看相册 / 系统分享 / 重新整理
+分享成功：留存本页，Toast「分享完成」+ 新增【回到首页】快捷按钮 NEW
+分享取消：Toast「分享已取消」；系统异常失败：Toast「分享失败，请重试」NEW
+「重新整理」双分支：有权限 → 直接进扫描页；权限失效 → 跳权限页 NEW
●固定入口：【新增宝宝】→ 宝宝资料页
↺ 「重新整理」→ 有效权限：直接进扫描页 | 权限失效：先落权限页，重新授权后进扫描页
全局特殊路径
●老用户冷启动（已授权 + 有数据）：直接跳结果页，跳过全部引导
●部分授权：全链路 Banner 透传，冷启动保留，权限恢复全量后自动消失
●拒绝权限冷启动（有历史数据）：仍停权限页，展示弹窗常驻去设置按钮，无法往下走
●中途系统关闭权限：从任意页面切回 App，弹窗拦截跳转权限页
●新增宝宝闭环：结果页/保存页「新增宝宝」→ 资料页（昵称必填，草稿自动回填）→ 保存后独立扫描 → 独立 Top 20
●多宝宝 + 混合权限：A 宝宝全授权、B 宝宝部分授权，切换宝宝同步切换 Banner 提示
●引导三类断点：权限中断落权限页 / 资料填写中断落资料页 / 扫描中断落扫描页

权限全链路规则表

三种权限状态 × 冷启动 / Banner / 退出重进 完整矩阵

权限类型冷启动行为全链路 Banner退出重进规则
完全授权 有数据直跳结果页 无任何提示 Banner 任意退出，冷启动停留在离开页面
部分授权 有数据直跳结果页 全页面顶部固定黄色 Banner，不可手动关闭，权限恢复后自动消失 冷启动保留 Banner，权限不变不消失
拒绝权限 有数据仍停权限页 弹窗常驻去设置按钮，无法往下走 每次冷启动强制拦截在权限页

评分算法

Vision 框架 · 规则评分 · iOS 16+ · 无需外部模型

维度权重实现方式
人脸 / 宝宝置信度 30% VNDetectFaceRectanglesRequest · 人脸大小占画面比例Vision
清晰度 25% 拉普拉斯方差（缩略图计算），严重模糊直接过滤
表情 / 眼睛状态 20% VNDetectFaceLandmarksRequest · 眼睛开合、嘴角上扬Vision
构图完整度 15% 人脸不超出画面边缘、主体居中比例
场景意义 / 连拍惩罚 10% 拍摄时间差 < 2s 视为连拍，同组最多保留 2 张，其余递减扣分
扫描范围：最近 90 天 与最近 1000 张 取并集（两个条件满足其一即纳入，上限 1000 张）
7 步流程：读取照片 → 过滤截图/二维码/视频 → 分批 50 张计算清晰度 → VNDetectFaceRectanglesRequest 过滤无人脸 → VNDetectFaceLandmarksRequest 获取表情分 → 连拍分组 → 综合评分排序取前 20

交互体验细节

用户体感升级 · 全局通用规则

💬
弹窗统一交互
点击空白区域可关闭弹窗，权限强制弹窗除外（必须二选一）
✨
按钮状态动效
条件不满足时按钮置灰；点击有轻弹反馈；不禁用按钮，改为触发内联错误提示
💾
全链路状态记忆
未点最终「保存精选」前，所有扫描结果、筛选状态、宝宝信息草稿全部本地缓存
🔒
隐私合规三锚点
启动页底部 / 权限页底部 / 设置页，三处可查看完整隐私协议，满足 App Store 审核
🔄
新手引导重置
设置内提供「重新走新手引导」开关，老用户主动回启动说明页
👶
多宝宝切换缓存
切换宝宝时保留各自的移除/精选状态，不重置，无感切换
